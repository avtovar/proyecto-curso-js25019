Analista QA

Control de calidad y testing automatizado/manual.

Pensum de Estudio para Tester QA

El pensum de un tester QA (Quality Assurance) suele abarcar tanto conocimientos teóricos como prácticos...

  • Fundamentos de Calidad del Software: Tipos de pruebas, SDLC, principios.
  • Herramientas de Testing: Selenium, JMeter, Cypress, etc.
  • Gestión de Defectos: Jira, Bugzilla.
  • Diseño de Casos de Prueba: Casos efectivos.
  • Planificación de Pruebas: Estrategias y criterios de aceptación.
  • Pruebas no Funcionales: Rendimiento, seguridad, etc.
  • Automatización de Pruebas: Scripting.
  • Programación y BD: Java, Python y SQL.
  • Comunicación: Trabajo en equipo.
  • Habilidades Blandas: Autonomía, adaptabilidad.

Certificaciones:

  • ISTQB: Fundamentos, técnicas y avanzadas.
  • Certificación en herramientas: Certificaciones específicas.

Analista Funcional

Relevamiento, documentación y diseño funcional.

Pensum de Estudio para Analista Funcional

  • Módulo 1: Introducción al análisis funcional o análisis de negocio.
  • Módulo 2: Elicitación y documentación de requisitos.
  • Módulo 3: Identificación de MVP, generación de roadmap y backlog.
  • Módulo 4: El rol del analista en entornos ágiles como Scrum.

Bases de Datos

MySQL, SQL Server, diseño relacional, normalización.

Pensum de Estudio para Bases de Datos

  • Diseño y diagramación de bases de datos: Modelado entidad-relación, normalización y relaciones.
  • Uso de inteligencia artificial: Aplicación de IA para generar y optimizar consultas SQL automáticamente.
  • Instalación y utilización de MySQL: Instalación local, herramientas gráficas como Workbench.
  • Estructuras de consultas SQL: Uso de operadores, cláusulas WHERE, GROUP BY, JOINs y sentencias básicas.

Scrum y Agile

Metodologías ágiles para gestión de proyectos.

Pensum de Estudio para Scrum y Metodologías Ágiles

  • Introducción a Scrum y metodologías ágiles: Principios, valores del manifiesto ágil, roles y artefactos en Scrum.
  • Liderazgo de equipos ágiles: Estilos de liderazgo adaptativos, facilitación de equipos autoorganizados.
  • Priorización de tareas: Técnicas como MoSCoW, matriz de valor, y uso del backlog.
  • Adaptación a cambios: Ciclos iterativos, revisiones, retrospectivas y gestión del cambio.
  • Mejora continua en proyectos: Aplicación del Kaizen, feedback constante y métricas de mejora.

Testing Automatizado

Selenium, Cypress, CI/CD y buenas prácticas.

Pensum de Estudio para Testing Automatizado

  • Automatización de pruebas en aplicaciones web: Introducción a testing automatizado, diferencias con pruebas manuales.
  • Uso de Selenium WebDriver, TestNG y Page Object Model: Creación de scripts robustos y reutilizables, estructuras de prueba escalables.
  • Generación de reportes y manejo de datos: Reportes HTML, uso de archivos externos como Excel/CSV para datos de prueba.
  • Integración con GitHub y ejecución por línea de comandos: Control de versiones, automatización con scripts en CI/CD.
